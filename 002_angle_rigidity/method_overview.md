# Method Overview

The reproduction follows the paper's computational path.

First, build the angle rigidity matrix. For a selected set of ordered triplets, each triplet contributes one row to `R_a(p)`. If `R_a(p)` reaches rank `2N - 4`, then the angularity is infinitesimally angle rigid.

Second, simulate the angle-only controller. For each agent `i`, the control input is a sum over angle constraints centered at `i`:

```text
u_i = - sum (alpha_jik - alpha_jik*) (z_ij + z_ik)
```

Here `z_ij` is the bearing from agent `i` to agent `j`. The control law moves each agent along the bisector-like direction generated by the two visible neighbors in each angle constraint.

Third, compare with the bearing-based controller:

```text
p_dot_i = - sum P_zij z_ij*
```

This comparison is useful because desired bearings are frame-dependent. If one local frame is rotated but the controller still uses the desired global bearing directly, the bearing controller can converge to the wrong shape. The angle controller avoids that issue because the angle error is the same in every rotated local frame.
